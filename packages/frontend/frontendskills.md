## ADHD-Friendly Design Principles

### Reduce Cognitive Load

-   One primary action per screen.
-   Avoid clutter and unnecessary UI elements.
-   Break large tasks into small steps.
-   Keep instructions short and visual.
-   Always show clear progress.

### Child-Friendly UI

-   Large rounded buttons.
-   Friendly typography.
-   Bright but calming colors.
-   Consistent spacing.
-   Large click/touch targets.

### Motivation

-   Progress bars.
-   XP and level indicators.
-   Stars, badges, and achievements.
-   Positive feedback after every success.
-   Celebrate completed tasks with subtle animations.

### Accessibility

-   WCAG 2.2 compliant.
-   Keyboard accessible.
-   High color contrast.
-   Semantic HTML.
-   Clear focus indicators.
-   Readable font sizes.

### Motion

Use animation intentionally:

-   Smooth page transitions.
-   Progress animations.
-   Success celebrations.
-   Loading skeletons.

Avoid excessive flashing or distracting motion.

------------------------------------------------------------------------

## React Best Practices

-   Prefer functional components.
-   Use TypeScript everywhere.
-   Keep components under 200 lines.
-   Build reusable UI components.
-   Separate UI, hooks, and API logic.
-   Handle loading, empty, and error states.
-   Prefer composition over prop drilling.
